AI Troubleshooting — v0.2
For this version I used AI as a Socratic learning partner rather than a source of code. Instead of handing me solutions, it asked guiding questions that pushed me to reason through problems myself, then flagged mistakes and explained the "why" so I understood each fix rather than just copying it. I wrote every line of code myself.
Compared to v0.1, this version needed less heavy debugging, as my Python foundation from Course 7 and the previous build was more solid. The main reasoning points I worked through were:

File handling — resolving a FileNotFoundError caused by an incorrect relative path and spaces in the filename. I learned to read the error output, check the file's true name and location, and avoid spaces in filenames going forward.
Adapting to real data — realising my old ERROR/WARNING conditionals wouldn't match a real SSH auth log, and reasoning out the correct vocabulary (Failed password, Disconnecting, error) by reading the actual log format.
Choosing the right data structure — weighing separate variables against a dictionary for storing counts, and reasoning that a dictionary was cleaner and better suited to tallying multiple event types.

Overall, AI supported my thinking without replacing it, which kept the learning and the code genuinely my own.
